# TODO 卸货到 prd/（归档已完成）

`TODO.md` 是**待办队列**；`prd/` 是**已实现产品文档**。TODO 过长时，应把**已稳定完成**的部分写入对应 `prd/` 模块，再从 TODO 精简，而不是无限堆 `[x]`。

与 **`update-prd` 全量重建** 的区别：

| | TODO 卸货（本规范） | update-prd 全量 |
| --- | --- | --- |
| 频率 | TODO 过长或功能块已验收闭环时 | 里程碑 / 03 收尾 / 大版本 |
| 范围 | 选定已完成的 §2.x / 章节 | 整个项目对照代码重建 |
| TODO | 删除或折叠已归档条目 | 主要写 §10 差距 |
| prd/ | **增量**更新相关模块 | 可读代码全量刷新 |

日常卸货用本规范；结构大变或首次迁移仍用 `update-prd` 全量。

## 何时卸货

满足**任一**即建议执行（`project-iterate` 末轮或用户明确要求）：

- `TODO.md` 超过 **~180 行**（或 §2 已完成的 MVP 功能 ≥ **2 个**且均 `[x]` 并过 project-verify）
- 某 **§2.x 功能块**全部 checkbox `[x]`，且对应验收已在 project-verify / 验证记录中通过
- 用户说「TODO 太长了」「归档一下完成的」
- 上下文将满：优先卸货**已闭环**模块，再写开发进度

**不要卸货：**

- 仍 `[ ]` 或有阻塞/未验收的条目
- 仅代码写完、TODO 未 `[x]` 或未验证
- 整个 `prd/` 尚不存在且无根目录 `prd.md` → 先走 `03` + `update-prd` 迁移

## 卸货流程

### 1. 选定归档块

从 TODO 选出**已闭环**块，例如：

- `## 2. MVP 功能拆解` 下某个 `### 2.x <功能名>`（含子功能、验收标准均已 `[x]`）
- 或 §3–§8 中整节已完成且与单一 prd 模块对应

记录：原章节编号、功能名、完成日期、验证记录引用（如有）。

### 2. 映射 prd 文档

| TODO 内容 | 写入 prd/ |
| --- | --- |
| 功能名、用户目标、入口 | 该模块 **功能概述** + **核心功能列表**（标为已实现） |
| 子功能 `[x]` 列表 | **核心功能列表** 子项，注明实现方式 |
| 主流程、状态 | **业务逻辑**（可 mermaid） |
| 数据需求 | **数据结构** |
| 架构/模块（§4 相关） | **相关代码文件** + 关联模块 |
| §2 验收标准（已通过） | 模块内 **已实现验收**（非待办 checkbox） |

- 已有 `prd/<模块>.md` → **增量更新**，不覆盖 unrelated 内容
- 无对应文档 → 新建 `prd/<模块-slug>.md`，并在 `prd/README.md` 索引中追加链接

### 3. 写入 prd 的「已实现」格式

在每个模块文档中追加或更新：

```markdown
## 已实现（自 TODO 归档）

- **归档日期**：YYYY-MM-DD
- **原 TODO 位置**：§2.1 模板写作
- **验证**：project-verify 验证记录 YYYY-MM-DD / 或本轮 lint+走查

### 实现摘要
- （3～5 条：用户可见能力 + 关键工程决策）

### 已实现验收
- 模板写作：选模板 + 主题 → 1～3 条结果 → 复制（已通过）
- …

### 相关代码文件
- `app/...`
- `lib/...`
```

`prd/` 描述必须与**当前代码一致**；写入前快速核对相关文件是否存在。

### 4. 精简 TODO

对已归档块，**二选一**（推荐 A）：

**A. 折叠为一行索引（推荐）**

```markdown
### 2.1 模板写作
- [x] **已归档** → 见 `prd/template-writing.md`（2026-05-29）
```

**B. 整节删除**

- 从 TODO 删除 §2.1 全文
- 在 **`## 已完成模块（详见 prd/）`**（02 模板 §1 之后）追加索引行：

```markdown
## 已完成模块（详见 prd/）

- 模板写作 → `prd/template-writing.md`（归档于 2026-05-29）
- 自由写作 → `prd/free-writing.md`（归档于 2026-05-29）
```

**保留在 TODO：**

- 所有 `[ ]`、进行中、阻塞项
- §10 未完成目标
- 开发进度 / 验证记录（最近 2～3 次即可；更早的可摘要后删）

### 5. 写回卸货记录

在 TODO 文末追加（可与开发进度合并）：

```markdown
## TODO 卸货记录

### 2026-05-29
- 归档：§2.1 模板写作 → `prd/template-writing.md`
- TODO 行数：210 → 142
- 下轮活跃项：§2.2 自由写作、§2.4 历史记录
```

## 调用方式

| 场景 | 做法 |
| --- | --- |
| 日常 1～2 个功能闭环 | `project-iterate` 结束前按本规范卸货，**不必**全量 update-prd |
| 多模块 / 架构大变 | 本规范卸货后，或另开一轮 **`update-prd`** 全量核对 |
| 仅有 `prd.md` 无 `prd/` | 不用本规范；走 `update-prd` 迁移 |

可 spawn 无专属 agent；主会话或 `project-iterate` 执行，必要时读 `update-prd` 的 PRD 章节标准。

## 自检

- [ ] 仅归档已 `[x]` 且已验证（或用户确认接受）的块
- [ ] `prd/` 已写入实现摘要、验收与代码路径，与代码一致
- [ ] `prd/README.md` 已更新索引（如有新模块）
- [ ] TODO 已折叠或删除对应块，并留索引或卸货记录
- [ ] 未把 `[ ]` 未完成项写进 prd「已实现」
