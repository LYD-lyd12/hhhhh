# TeleToken Router 设计系统

## 设计取向依据

TeleToken Router 是面向企业和开发者的 AI 模型聚合中台，核心用户是技术总监和独立开发者，需要体现专业、可靠、高效的企业级中台风格。设计应注重数据可视化、操作便捷性和系统稳定性的视觉传达。

### 用户画像适配
| 维度 | 来自 prd.md 的要点 | 对应设计决策 |
| --- | --- | --- |
| 用户熟练度 | 技术总监和独立开发者，熟悉技术术语 | 使用专业术语，减少引导性文案 |
| 使用频率 | 高频使用，日常工作工具 | 提供快捷操作入口，默认显示常用功能 |
| 注意力状态 | 单任务专注工作 | 简洁布局，减少视觉干扰 |
| 任务严肃度 | 企业级服务，涉及计费和服务稳定性 | 配色克制，错误态强调，数据精确展示 |
| 信息密度 | 需要查看大量监控数据和报表 | 紧凑列表布局，数据表格为主 |
| 情绪调性 | 专业工作场景 | 专业型空态，强调效率和可靠性 |
| 核心转化动作 | API 接入、模型订阅、查看账单 | 主 CTA 突出，流程清晰 |

## UI 技术栈与组件策略
- **样式**：Tailwind CSS + CSS variables（全局样式入口：`@import "tailwindcss"`）
- **组件**：默认 Tailwind 组合；复杂 Dialog / Dropdown / Select / Tabs 等采用 shadcn/Radix 风格
- **图表**：recharts（用于用量看板、趋势图、节点状态监控）
- **禁止**：独立 CSS 文件堆叠、CSS-in-JS、用 emoji 充当功能图标

## 图标 Iconography
- **图标库**：`lucide-react`（全站唯一）
- **安装包名**：`lucide-react`
- **尺寸 token**：导航 `--icon-nav` 20px；按钮内 `--icon-inline` 16px；空态/插画区 `--icon-empty` 48px
- **线宽 / 风格**：stroke 1.5，与字重一致；禁止多套线宽混用
- **无障碍**：纯图标按钮必须 `aria-label` 或 sr-only 文案；导航项图标+文字
- **动作映射**：主导航使用 Settings、BarChart3、Package、Users 等语义；保存用 Save、删除用 Trash2、关闭用 X

## 媒体与占位图
- **默认策略**：`public/placeholders/` 静态资源 + SVG 占位组件；不默认依赖海外 CDN
- **Unsplash API / 远程图源**：无（企业级中台不需要图库/摄影内容）
- **尺寸与 a11y**：固定宽高比、必有 `alt`、加载失败显示本地 fallback，避免 CLS
- **按界面**：头像用初始字母头像组件、封面用渐变色块、卡片图用图标占位

## 数据与演示内容（设计侧约定）
- **数据来源类型**：`LIVE` | `FIXTURE` | `EMPTY`
- **演示策略类型**：`LIVE_UNCONFIGURED` | `FIXTURE_DEMO` | `EMPTY_CTA`
- **FIXTURE 默认路径**：`src/fixtures/`
- **FIXTURE 是否显示「示例数据」角标**：是
- **LIVE 未配置态**：文案与 CTA 基调专业、简洁

## 视觉 Token
- **主色调**：蓝色系（#1e40af，代表科技感和可靠性）
- **辅助色**：绿色（#16a34a，成功状态）、橙色（#f97316，警告状态）、红色（#dc2626，错误状态）
- **字体**：Inter，14px 正文，16px 标题，字重 400-600
- **圆角**：4px 按钮/输入框，8px 卡片，12px 弹窗
- **阴影**：低对比度阴影，hover 时增强
- **间距**：8px 基础单位，模块间距 16-24px

## 组件气质与关键界面状态
- **默认态**：清晰的信息层级，重点突出
- **空态**：专业提示文案 + 操作引导按钮
- **加载态**：骨架屏或 spinner，避免空白
- **成功态**：绿色提示 + 明确反馈
- **错误态**：红色警告 + 错误详情 + 解决方案建议
- **未配置态**：引导配置的 CTA 按钮

## 响应式或窗口尺寸
- **桌面端**（≥1280px）：完整侧边栏 + 主内容区，多栏布局
- **平板端**（768px-1279px）：折叠侧边栏，自适应布局
- **移动端**（<768px）：底部导航，单列布局
